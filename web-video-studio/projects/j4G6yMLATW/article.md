# 告别繁琐插件，一行代码让你的网页“动”起来
## 第一节：从 Flash 时代到 HTML5 视频革命 

不知道你是否还记得，十几年前想在网页上看个视频，浏览器总要弹出一个提示：“需要安装 Flash 插件”。那个年代，视频播放是插件的天下，不仅加载慢、占内存，还动不动就崩溃。手机端更是直接告别 Flash，想看个视频得跳转到专门的 App。直到 HTML5 标准横空出世，一切才彻底改变。它给网页带来一个原生的 `<video>` 标签，就像插入图片一样简单。你不再需要任何第三方软件，浏览器自己就能流畅解码、播放视频。这个标签背后，是整个互联网向开放、高效、安全迈出的一大步。而对于我们普通创作者来说，用 HTML 做视频，其实比你想象中要简单一百倍。接下来，我就带你一步步上手，从零开始，把视频优雅地放进你的网页里。

## 第二节：一个标签，让视频立刻出现 

先来点实际的。打开一个空白文本文档，保存成 `.html` 后缀，写入下面这段代码，然后用浏览器打开它：

```html
<video src="my-video.mp4" controls width="600"></video>
```

就这样，一个带播放按钮、进度条、音量调节的视频播放器就诞生了。`src` 属性是视频文件的路径，可以是本地文件，也可以是一个网上的链接。`controls` 这个单词一加，浏览器就会自动配上那一整套默认的控件：播放/暂停、时间显示、拖动进度、音量、全屏。`width` 用来设定宽度，高度会自动按比例缩放。如果你希望视频加载后自己就开始播放，就再加一个 `autoplay`；如果想让它播完从头再来，就写上 `loop`；想默认静音，避免自动播放被浏览器拦截，就加上 `muted`。这些属性全可以组合使用，比如：

```html
<video src="demo.mp4" autoplay muted loop controls width="100%"></video>
```

这样就能得到一个自动循环播放、静音、并且宽度撑满容器的背景视频。是不是感觉跟搭积木一样？但 `<video>` 标签的魔法才刚刚开始。

## 第三节：打破格式壁垒，让所有浏览器都听话 

现实很骨感：浏览器之间对视频格式的支持并不统一。你可能会遇到这样一种情况——兴致勃勃用 Chrome 打开，视频流畅播放，发给用 Safari 的朋友却是一片黑屏。这是因为 `.mp4`（H.264 编码）虽然兼容性最好，但有些浏览器更偏爱 `.webm`（VP8/VP9），或者 `.ogg`（Theora）。要解决这个问题，`<video>` 标签支持一种“多重保险”的写法：把同一个视频转成多个格式，然后让浏览器自己挑它能播的那个。像这样：

```html
<video controls>
  <source src="my-video.mp4" type="video/mp4">
  <source src="my-video.webm" type="video/webm">
  <source src="my-video.ogv" type="video/ogg">
  你的浏览器不支持视频播放，请升级浏览器。
</video>
```

在 `<video>` 和 `</video>` 之间，我们不再用 `src` 属性，而是放入多个 `<source>` 标签。每个 `<source>` 提供一种格式，浏览器会从上到下依次尝试，选中第一个它能解码的格式。如果三种格式都不支持，最后那一行文字就会显示出来，相当于一个友好的兜底提示。这样，你就能做到一个视频走遍天下，再也不用为格式而发愁。而且，`type` 属性的作用是提前告诉浏览器文件的视频编码，省得它自己去下载一段再检测，加载速度会更快。

## 第四节：不止于默认控件，做一套属于你自己的播放器 

默认的控件虽然方便，但长相千篇一律，而且在不同浏览器上外观还略有差别。如果你想做出个性化十足的播放器——比如一个圆形的播放按钮、自定义的进度条颜色、甚至添加快进快退——那就必须关掉默认控件，自己用 HTML、CSS 和 JavaScript 重新造一个。

第一步，先加上 `controls` 属性没关系，但我们要替换它，就可以去掉，或者保留它也可以隐藏，但为了纯自定义，我们这样做：

```html
<video id="myVideo" src="movie.mp4" width="640"></video>
```

然后我们用几个按钮和 `<div>` 模拟控件。

```html
<button id="playPauseBtn">播放</button>
<input type="range" id="progressBar" value="0" min="0" max="100" step="0.1">
<span id="timeDisplay">00:00 / 00:00</span>
```

接下来用 JavaScript 把逻辑串起来。当点击按钮时，检查视频是暂停还是播放，然后切换状态，并把按钮文字改成“暂停”或“播放”。进度条可以通过监听视频的 `timeupdate` 事件实时更新，同时用户拖动进度条时也可以用 `video.currentTime` 控制跳转。时间显示则利用视频的 `currentTime` 和 `duration` 属性格式化成分秒。这样一来，你就获得了一台属于自己的视频播放器，完全按照你的设计风格来展现。想要加上倍速播放？只需要一个 `video.playbackRate = 1.5` 就能搞定。想实现画中画？调用 `video.requestPictureInPicture()` 即可。浏览器的 API 已经非常丰富，创造空间巨大。

## 第五节：让视频开口说话——字幕、章节与无障碍 

一个好的视频体验不能缺少字幕。对于听障人士，或者需要在静音环境下观看的用户来说，字幕就是视频的另一张嘴。HTML5 提供了 `<track>` 标签，专门用来加载外挂字幕文件。最常见的字幕格式是 WebVTT（ `.vtt` 文件），基本长这样：

```
WEBVTT

00:00:01.000 --> 00:00:03.000
大家好，欢迎收看今天的视频。

00:00:03.500 --> 00:00:06.000
今天我们来聊聊如何用HTML做视频。
```

把这个文件保存好，然后在 `<video>` 标签里加上：

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track label="中文" kind="subtitles" srclang="zh" src="captions-zh.vtt" default>
  <track label="English" kind="subtitles" srclang="en" src="captions-en.vtt">
</video>
```

`kind` 可以设为 `subtitles`（字幕）或 `captions`（包含环境音的隐藏式字幕），`srclang` 标明语言，`label` 是选项显示的名字。加上 `default` 就会默认启用这条字幕。用户可以通过播放器上的 CC 按钮自由切换语言。另外，`<track>` 还可以用于章节导航（`kind="chapters"`），这样在进度条上会显示出章节标记，点击即可跳转，非常适合长视频教程或播客。你看，视频的交互性和包容性，一下子就上了一个台阶。

## 第六节：让视频适应任何屏幕，并顺利“出海” 

现在绝大多数流量都来自手机。如果你的视频宽度固定为 600 像素，在电脑上看着正好，在手机上就可能超出屏幕，或者被压得只剩一条缝。响应式设计必须安排上。最简单的方法是设置 `max-width: 100%` 和 `height: auto`，通过 CSS 就能让视频跟随父容器自适应。比如：

```css
video {
  max-width: 100%;
  height: auto;
}
```

如果你想实现一个 16：9 的容器，让任何尺寸的视频在里面居中裁剪或填充，可以用经典的 “padding-top 百分比” 技巧制作一个等比例盒子，然后把视频绝对定位放进去。这部分网上有很多代码模板，直接套用即可。

除了尺寸，性能优化也很关键。我们通常会给视频加上 `preload` 属性，`preload="metadata"` 表示只预加载视频的基本信息（如时长、第一帧），而不会下载整个视频，节省用户流量；`preload="none"` 则是完全不预加载，等用户点击播放才下载。另外，别忘了给 `<video>` 标签加一张海报图 `poster="cover.jpg"`，这样在视频加载前或者未播放时，显示一张精美的封面，而不是一片黑漆漆的画面，用户体验立刻提升。还有一个常见问题是，iOS 系统在低电量模式下可能会禁止自动播放带声音的视频，所以 `muted` 属性往往是自动播放成功的“通行证”。

## 第七节：总结与脑洞：视频+HTML的无限可能 

现在我们回顾一下：你学会了用 `<video>` 标签一行嵌入视频，用多个 `<source>` 解决兼容性，用属性控制播放行为，用 JavaScript 打造自定义控件，用 `<track>` 加字幕，用 CSS 做响应式。这些看似简单的积木块，一旦组合起来，就能做出各种酷炫的效果。比如网站背景全屏播放静音视频，配合文字 overlay 打造大片感；比如用视频作为产品介绍，鼠标悬停才播放；甚至结合 Canvas 做实时视频特效。HTML 不再只是一个标记语言，它已经变成了连接媒体、交互和设计的重要纽带。

当然，视频文件本身的大小也要考虑，通常推荐用 H.264 编码的 MP4，兼顾画质和体积，再配合 CDN 加速分发。如果你想做直播，还可以研究一下 HLS 协议与 `<video>` 的配合。一句话，用 HTML 做视频，门槛极低，但天花板却很高。希望今天的分享能帮你推开这扇门，让视频成为你网页表达中最生动的一部分。下一次，当你在网上看到任何一段视频时，不妨右键检查一下，说不定就能看到我们今天讲到的这些熟悉标签。去尝试，去创造，让你的网站真正动起来吧！

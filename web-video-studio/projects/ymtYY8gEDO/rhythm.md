# 节奏蓝图

> 总时长：约 320s（~5 分 20 秒）· 情绪弧：质疑 → 铺垫 → 颠覆 → 证明 → 行动（论证型）

## 节拍规划

| # | 类型 | 时长 | 情绪目标 |
|---|------|------|---------|
| 1 | [hook] | ~12s | 悬念：深夜改 Bug 的痛苦谁没经历过？要是终端里有个帮手直接把活儿干了呢？ |
| 2 | [build] | ~80s | 铺垫：Claude Code 是什么东西，它不只是补全代码——它是一个住在终端里、会自己动手的执行者 |
| 3 | [turn] | ~45s | 颠覆：你以为是另一个 Copilot？完全不是——一个是输入法增强，一个是坐在你旁边的同事（全文 35% 位置） |
| 4 | [data] | ~70s | 数据实锤：感知-行动循环拆解——它怎么读代码、改文件、跑测试、自己迭代的 |
| 5 | [release] | ~18s | 释放：安全问题？有护栏——每一步你都看得见、管得住 |
| 6 | [build] | ~80s | 再铺垫：一个真实场景演示 + 开发者角色进化——从工匠到指挥官 |
| 7 | [close] | ~15s | 收束：打开终端试试，你会爱上这种感觉 |

# 节奏设计指引（Rhythm Design）

在写 script.md 之前，必须先完成节奏蓝图。这是"情绪编排"前置于"信息翻译"的核心机制。

---

## 为什么需要节奏蓝图

AI 默认按逻辑顺序平铺信息，节奏是信息量的结果，是被动的。
节奏蓝图让 AI 先设计**观众的注意力走向**，再往每个位置填信息——节奏主动，信息服从。

观众的注意力曲线不是线性的，是"高度紧绷 → 短暂释放 → 再次紧绷"的波浪形。
平铺的信息流会让大脑预判并"关掉"摄入。节奏蓝图的作用是强制制造波浪。

---

## rhythm.md 格式

```markdown
# 节奏蓝图

> 总时长：约 <T>s · 情绪弧：<弧线名称>（见下方模板库）

## 节拍规划

| # | 类型 | 时长 | 情绪目标 |
|---|------|------|---------|
| 1 | [hook] | ~Xs | 悬念：<一句话目标> |
| 2 | [build] | ~Xs | 铺垫：<一句话目标> |
| 3 | [turn] | ~Xs | 颠覆：<一句话目标>（全文 30–40% 位置） |
| ... | ... | ... | ... |
```

**约束：**
- `[turn]` 节拍必须落在全文总时长的 30–40% 处（第二次钩子，最影响完播率）
- `[release]` 节拍必须落在信息最密集的 `[data]` 之后
- 总节拍数 ≥ 5，≤ 8
- 所有节拍时长之和 ≈ 视频目标总时长（误差 < 10%）

---

## 节拍类型定义

| 类型 | 情绪目标 | 信息密度 | 对应章节视觉策略 |
|------|---------|---------|----------------|
| `[hook]` | 制造悬念，打破惯性认知 | 极高，每步 ≤ 1 句 | 大字 + 极简背景，快切，不堆信息 |
| `[build]` | 堆积认知，建立预期 | 中高，节奏匀速 | 信息逐层叠加，Stagger 入场 |
| `[turn]` | 颠覆预期，第二次钩子 | 高，强情绪 | 视觉反转：已建图表被划掉 / 左右对比布局切换 |
| `[data]` | 数据实锤，建立可信度 | 高，数字密集 | 必须有 Counter / 折线 / 柱状等可视化动画 |
| `[release]` | 给大脑喘息，情绪释放 | 极低，1 句话 | 黑底白字单句 / 反常识数字停顿 / 大号留白 |
| `[close]` | 快速收束，留下钩子 | 低，记忆点强 | 收束 + 悬念句 / 行动指令 |

---

## 情绪弧模板库

根据文章类型选择最合适的弧线，**不要凭空设计**，先套模板再微调：

### 论证型（为什么 X / 反常识解析）
```
情绪弧：质疑 → 铺垫 → 颠覆 → 证明 → 行动
节拍序列：[hook] → [build] → [turn] → [data] → [release] → [close]
[turn] 位置：全文 35%
典型触发词：大家以为 / 其实 / 你可能不知道 / 真相是
```

### 故事型（某人某事 / 案例叙事）
```
情绪弧：共鸣 → 冲突 → 转折 → 解脱 → 启示
节拍序列：[hook] → [build] → [turn] → [release] → [build] → [close]
[turn] 位置：全文 40%
典型触发词：有一天 / 结果发现 / 但他没想到 / 这件事让我意识到
```

### 方法论型（怎么做 X / 实操教程）
```
情绪弧：痛点 → 旧方法失败 → 新方法揭示 → 演示 → 呼吁
节拍序列：[hook] → [data] → [turn] → [build] → [release] → [close]
[turn] 位置：全文 30%
典型触发词：以前大家都 / 但这个方法有个问题 / 换一种思路 / 试试这个
```

### 数据报告型（X 的现状 / 年终盘点）
```
情绪弧：反常识数字 → 背景 → 深挖 → 意外结论 → 预测
节拍序列：[hook] → [build] → [data] → [turn] → [release] → [close]
[turn] 位置：全文 40%
典型触发词：这个数字 / 你没想到吧 / 但背后的原因 / 这意味着
```

### 对比型（X vs Y / 选择指南）
```
情绪弧：假设 → 对比建立 → 意外胜负 → 原因分析 → 选择建议
节拍序列：[hook] → [build] → [turn] → [data] → [close]
[turn] 位置：全文 35%
典型触发词：看起来 A 更好 / 但实际上 / 测下来结果让人意外 / 所以选
```

## rhythm.md 自检（写完必走）

- [ ] `[turn]` 落在全文 30–40% 位置？
- [ ] `[release]` 在信息最密集节拍之后？
- [ ] 每段节拍时长合理（`[hook]` ≤ 15s，`[release]` ≤ 8s）？
- [ ] 总时长之和 ≈ 视频目标时长（误差 < 10%）？
- [ ] 情绪弧从模板库选择，有明确名称？

---

## 与 script.md 的衔接

rhythm.md 完成后，写 script.md 时：
- 每段口播按节拍类型写，`[hook]` 段必须极短极冲，`[release]` 段只允许 1 句
- `---` 分隔符在节拍切换点改写为 `---[类型]`，同类型内部的步骤分隔保持普通 `---`
- `[turn]` 段的第一句必须打破上一段建立的预期（不能顺着往下说）

## 与 outline.md 的衔接

outline 每章标题注明来自哪个节拍类型，例：
```
## 3. stableprice — 单价其实很稳（[data] · 6 steps · ~25s）
```

chapter agent 构建该章时，根据节拍类型约束视觉策略（详见节拍类型定义表）。
